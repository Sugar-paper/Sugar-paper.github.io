配置VS2019+QT的C++开发环境

发表于2020-02-25|更新于2020-03-07|QT C++
|阅读量:

安装QT

QT提供了QTCreater编辑器，请先确保使用它可以进行正常的编译与运行。[下载]
安装QT时在选择组件界面根据自己的需求选择QT版本，这里我们选择MSVC2017 64-bit版本。

安装VS

安装VS2019，自行到微软官网下载安装。

安装QTVS插件

待一切安装完成后，下载QT插件安装。

开发版插件
稳定版插件

配置路径

打开VS->继续但无代码->扩展->QTVSTool->QTOption。
点击Add加上下载的版本。

测试

建立一个带UI的空项目，能正常编译运行，显示程序窗口就成功了。

踩过的坑

  • error MSB8036: 找不到 Windows SDK 版本【版本号10.xxx】…
  • 如果没有勾选就打开VSInstaller修改一下安装。
  • 如果有勾选并在项目中能看到WindowsSDK的版本，那么应该是VS安装过程出现的问题，打开VSInstaller修改，取消勾选的版本并勾选 另一个版本再次安装即可。
  • fatal error C1083: 无法打开包括文件:“stddef.h”: No such file or directory…
  • 在项目的Include中添加有此头文件的路径。
  • 上面继续报错且文件改变，那么可以肯定的是在安装VS时导致的文件释放或编译出错，那么恭喜你只能卸载重新安装VS，并且通过修复并不会成功，至少我是这样的。
  • 中文显示乱码
  • 在头文件中添加
  • qDebug在控制台无显示
    右键项目，属性，链接器，系统，把默认的，窗口 (/SUBSYSTEM:WINDOWS)改为 控制台 (/SUBSYSTEM:CONSOLE)
